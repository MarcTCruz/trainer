## Anti-Patterns Verdict

LLM assessment: The shell reads competent but cold — dark cyberpunk/terminal against a stated "playful, encouraging" brand (PRODUCT.md anti-reference #1 is LeetCode, which this resembles). Carried from the 2026-06-23 critique and not visibly addressed.

Deterministic scan (detect.mjs on index.html): 1 finding — `single-font` (Fira Code). False positive: a single monospace identity is intentional for a code trainer. The CSS-level June-23 bans (gradient text, side-stripe borders, bounce easing in style.css) are NOT visible to the markup detector and were NOT re-verified this run — treat as open until checked.

## Overall Impression

Two regressions since June-23, both from features added AFTER that critique and never re-reviewed: (1) a whole learning path (Euclid's Elements) is shipped but unreachable, and (2) the data/sharing controls violate honest-labeling and destructive-action-scope. The single biggest opportunity: make every built path reachable and make destructive/visibility actions tell the exact truth.

## What's Working

1. Editor-as-center-of-gravity matches design principle #1; run loop is fast and legible (expected/got test cards).

3. Token architecture is consistent.

## Priority Issues

### [P0] Euclid's Elements track is unreachable
The registry defines it (with Proposition 1) but `renderTrackRibbon` (src/app.js:365) hardcodes `getTrackExercises('thirty-days')`; there is no track selector. Zero presence in the rendered DOM. A built learning path the user cannot reach is a blocking IA failure. Fix: data-driven track selector iterating registry tracks. Suggested: /impeccable shape (nav) then build.

### [P1] "Clear All My Data" is mis-scoped and destructive
Currently deletes the CI fork AND clears all browser storage (token, progress, preferences) and signs out. Owner requirement: clear ONLY the public GitHub footprint, never touch the browser, and name exactly what it removes. Error-prevention failure. Fix: rename to the bounded action, hard-confirm, name the objects. Suggested: /impeccable clarify + harden.

### [P1] "Make Public" reflects a stale local flag, not real state
Reads a cached `REPO_PUBLIC_KEY` not live GitHub visibility; shows "Make Public" for an already-public repo, stays enabled, click is a silent no-op. Visibility-of-status failure. Fix: read live visibility, reflect state, give loading + success/failure feedback. Suggested: /impeccable harden.

### [P1] "Your Data" label is dishonest
Opens an info/policy panel, not the user's data. Match-real-world + help failure. Fix: rename to announce intent ("Storage Policy") and demote to a secondary info link. Suggested: /impeccable clarify.

### [P1] Personality / aesthetic mismatch (carried from 2026-06-23, unverified)
Cyberpunk-cold vs stated playful/encouraging; CSS absolute-ban violations from June-23 need re-verification in style.css. Suggested: /impeccable colorize + polish.
